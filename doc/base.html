Base

MLib est une petite librairie pour faire des interfaces graphiques utilisant pygame. Elle fonctionne avec un système de widget. Un widget est un élément graphique de la fenêtre. L'élément fenêtre est un widget un peu différent qui contient tout les autres widgets. Il y a un système de hiérarchie où les widgets peuvent avoir des parents. Le widget enfant s'affiche dans le widget parent et adopte son système de coordonnée. Les widgets ont une taille (w, h) et une position (x, y). On peut modifier leur apparence pixel par pixel. La fenêtre principale s'occupe d'actualiser l'interface graphique.

Besoins

Pour pouvoir s'éxécuter, MLib nécessite l'installation de la librairie pygame et pyperclip (elles sont importées avec la formule "import lib" et non "from lib import *").
Pour installer ces librairies, ouvrez l'invite de commande (la touche Windows+r puis marquer "cmd" sur windows) et tapez les 2 commandes suivantes :
pip install pygame
pip install pyperclip

Classes

MLib utilise la programmation orientée objet pour fonctionner. Chaque élément graphique correspond à une instance de classe (MWidget, MText...). Il y a 2 types de classes :
-Primaire : classe permettant de construire d'autre classes de même fonction sans besoins d'autre classes poru cette fonction
-N aire : classe hérité de classe N-1 aire

Évènements

Les évènements sont des actions qui sont effectuées soit par l'utilisateur, soit par un widget, soit par l'OS.
Voici une liste d'évènements pour la version 1.0.0 :
-pygame.QUIT : permet de quitter la fenêtre graphique.
-pygame.MOUSEBUTTONDOWN : un bouton de la souris est clické.
-pygame.MOUSEBUTTONUP : un bouton de la souris n'est plus clické.
-pygame.KEYDOWN : une touche du clavier est clické.
-pygame.KEYUP : une touche du clavier n'est plus clické.
-pygame.TEXTINPUT : du texte en entré par l'utilisateur.
