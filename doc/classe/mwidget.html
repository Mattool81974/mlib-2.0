MWidget

MWidget est la classe principale primaire pour définir un widget.
Elle hérite de MObject.
Ajouté à la version 1.0.0.

Fonctions

__init__()

Paramètres : x (assignée à la variabe x), y (assignée à la variabe y), width (assignée à la variable width), height (assignée à la variable height), parent (assignée à la variable parent), widgetType (assignée à la variable _type, basiquememnt mit à "MWidget")
Construit un objet widget.
Modifié à la version 2.3.0.

absolutePos()

Paramètres : aucun
Retourne un tuple avec (absoluteX(), absoluteY()).
Ajouté à la version 1.0.0.

absoluteX()

Paramètres : aucun
Retourne le x du widget par rapport au MApp (l'addition des coordonnées x des parents du widget).
Ajouté à la version 1.0.0.

absoluteY()

Paramètres : aucun
Retourne le y du widget par rapport au MApp (l'addition des coordonnées y des parents du widget).
Ajouté à la version 1.0.0.

containsChild()

Paramètres : enfant (widget à chercher)
Retourne -1 si l'enfant n'est pas dans la liste _children ou l'index si l'enfant y est.
Ajouté à la version 1.0.0.

delete()

Paramètres : aucun
Supprime un widget de l'environnement de MLib.
Attention, ne le supprime pas de la mémoire et du programme !
De plus, il est déconseillé d'appeler cette fonction en fin de programme (python s'en charge).
Ajouté à la version 2.0.1.

getBackgroundColor()

Paramètres : aucun
Retourne la couleur d'arrière plan backgroundColor du widget.
Ajouté à la version 1.0.0.

getCursorOnOverflight()

Paramètres : aucun
Retourne le curseur de survol cursorOnOverflight.
Ajouté à la version 1.0.0.

getHeight()

Paramètres : aucun
Retourne la hauteur du widget.
Ajouté à la version 1.0.0.

getIgnoreUserEvent()

Paramètres : aucun
Retourne la valeur du booléen qui indique si le widget ignore les évènements utilisateur.
Ajouté à la version 2.1.0.

getMouseDown()

Paramètres : aucun
Retourne le boutton clické sur le widget mouseDown.
Ajouté à la version 1.0.0.

getMouseUp()

Paramètres : aucun
Retourne le boutton clické sur le widget mouseUp.
Ajouté à la version 1.0.0.

getOverflighted()

Paramètres : aucun
Retourne si le widget est survolé par le curseur.
Ajouté à la version 1.0.0.

getParent()

Paramètres : aucun
Retourne le parent du widget.
Ajouté à la version 1.0.0.

getRect()

Paramètres : aucun
Retourne x, y, width et height sous la forme (x, y, width, height).
Ajouté à la version 1.0.0.

getShouldModify()

Paramètres : aucun
Retourne la valeur de shouldModify du widget.
Ajouté à la version 1.0.0.

getVisible()

Paramètres : aucun
Retourne la valeur de visible.
Ajouté à la version 1.0.0.

getWidth()

Paramètres : aucun
Retourne la largeur du widget.
Ajouté à la version 1.0.0.

getX()

Paramètres : aucun
Retourne la coordonnée X du widget.
Ajouté à la version 1.0.0.

getY()

Paramètres : aucun
Retourne la coordonnée Y du widget.
Ajouté à la version 1.0.0.

isMoving()

Paramètres : aucun
Retourne si le widget est en train de bouger (x ou y modifié) pendant cette frame.
Ajouté à la version 2.0.1.

isResized()

Paramètres : aucun
Retourne si le widget est en train de changer de taille (width ou height modifié) pendant cette frame.
Ajouté à la version 2.0.1.

move()

Paramètres : newX (nouvelle coordonnée X du widget), newY(nouvelle coordonnée Y du widget)
Change les coordonnées du widget X et Y à newX et newY.
Ajouté à la version 1.0.0.

posIn()

Paramètres : pos (coordonnées à tester)
Retourne si les coordonnées "pos" se situent dans le widget grâce à "absolutePos".
Ajouté à la version 1.0.0.

promoveChild()

Paramètres : child (enfant à privilégier)
Place child en haut de la liste de widget à render pour le render sur tous les autres.
Ajouté à la version 2.0.0.

resetWidget()

Paramètres : aucun
Reset le widget à son état initial (si aucun évènements ne le modifie).
Pour cela, la fonction utiliser toutes les variables (constantes) qui s'appellent "_NOMDUNEVARIABLE", comme _BACKGROUNDCOLOR.
Ajouté à la version 1.0.0.

resize()

Paramètres : newWidth (nouvelle largeur du widget), newHeight (nouvelle hauteur du widget)
Change la hauteur et la largeur du widget.
Ajouté à la version 1.0.0.

setBackgroundColor()

Paramètres : backgroundColor (nouvelle couleur d'arrière plan), constant (booléen qui définit si la modification est constante)
Change la couleur d'arrière plan "backgroundColor" du widget pour backgroundColor.
Si "constant" est True, "backgroundColor" et _BACKGROUNDCOLOR sont modifiés. Si il est à False, seulement "backgroundColor" est modifié pendant seulement une frame (le widget est stocké dans la liste _modifiedWidget de MApp).
Ajouté à la version 1.0.0.

setCursorOnOverflight()

Paramètres : cursorOnOverflight (nouveau curseur de survol)
Change le curseur de survol "cursorOnOverflight" en cursorOnOverflight.
Ajouté à la version 1.0.0.

setHeight()

Paramètres : newHeight (nouvelle hauteur du widget)
Change la hauteur du widget pour newHeight.
Modifié à la version 2.0.1.

setIgnoreUserEvent()

Paramètres : ignoreUserEvent (nouvelle valeur du booléen qui indique si le widget ignore les évènements utilisateur)
Change la valeur du booléen qui indique si le widget ignore les évènements utilisateur par "ignoreUserEvent".
Ajouté à la version 2.1.0.

setParent()

Paramètres : newParent (nouveau parent)
Change le "parent" du widget à newParent.
Ajouté à la version 1.0.0.

setVisible()

Paramètres : visible (nouvelle valeur de visible)
Change la valeur de visible par "visible".
Ajouté à la version 1.0.0.

setWidth()

Paramètres : newWidth (nouvelle largeur du widget)
Change la largeur du widget pour newWidth.
Modifié à la version 2.0.1.

setShouldModify()

Paramètres : shouldModify (nouvelle valeur de shouldModify)
Modifie shouldModify en configurant aussi l'état des parents.
Ajouté à la version 1.0.0.

setX()

Paramètres : newX (nouvelle coordonnée X du widget)
Change la coordonnée X du widget.
Modifié à la version 2.0.1.

setY()

Paramètres : newY (nouvelle coordonnée Y du widget)
Change la coordonnée Y du widget.
Modifié à la version 2.0.1.

softResetWidget()

Paramètres : aucun
Effectue un reset du widget comme resetWidget sans modifications graphiques.
Modifié à la version 2.0.1.

_addChild()

Paramètres : enfant (widget à ajouter en enfant)
Ajoute l'enfant "enfant" à la liste d'enfant _children et configure le tout.
Ajouté à la version 1.0.0.

_declaringWidget()

Paramètres : widget (widget à transmettre au parent)
Fonction permettant de passer le widget de parent en parent jusqu'au MApp.
Ajouté à la version 1.0.0.

_isGettingMouseDown()

Paramètres : button (nombre entier du bouton de souris clické), relativePos (position relative au widget du curseur de la souris)
Fonction appelé lorsque le widget est clické.
Fonction dédiée à l'héritage et à la réimplentation qui est appelé par MApp.
Si button est "1", la touche clické est la touche gauche, si button est "2", la touche clické est la molette, si button est "3", la touche clické est la touche droite.
Ajouté à la version 1.0.0.

_isGettingMouseUp()

Paramètres : button (nombre entier du bouton de souris clické), relativePos (position relative au widget du curseur de la souris)
Fonction appelé lorsque le widget n'est plus clické (après _isGettingMouseDown).
Fonction dédiée à l'héritage et à la réimplentation qui est appelé par MApp.
Si button est "1", la touche clické est la touche gauche, si button est "2", la touche clické est la molette, si button est "3", la touche clické est la touche droite.
Modifié à la version 1.3.0.

_isGettingOverflighted()

Paramètres : relativePos (position relative au widget du curseur de la souris)
Fonction appelé lorsque le widget est survolé.
Fonction dédiée à l'héritage et à la réimplentation qui est appelé par MApp.
La fonction est concue pour que les modifications ne sont valables qu'une frame, il est donc conseillé, en cas d'utilisation de fonction "setQuelqueChose" de passer "False" comme second paramètre.
Cette fonction ne s'occupe pas du curseur, c'est MApp qui s'en occupe.
Ajouté à la version 1.0.0.

_isKeyGettingDropped()

Paramètres : key (touche lachée)
Fonction appelé lorsque le widget est focus est qu'une touche du clavier est lachée.
Fonction dédiée à l'héritage et à la réimplentation qui est appelé par MApp.
La fonction est concue pour que les modifications ne sont valables qu'une frame, il est donc conseillé, en cas d'utilisation de fonction "setQuelqueChose" de passer "False" comme second paramètre.
Ajouté à la version 1.0.0.

_isKeyGettingPressed()

Paramètres : key (touche pressée)
Fonction appelé lorsque le widget est focus est qu'une touche du clavier est pressée.
Fonction dédiée à l'héritage et à la réimplentation qui est appelé par MApp.
La fonction est concue pour que les modifications ne sont valables qu'une frame, il est donc conseillé, en cas d'utilisation de fonction "setQuelqueChose" de passer "False" comme second paramètre.
Ajouté à la version 1.0.0.

_isMoving()

Paramètres : axis (axe x ou y modifié), relativeMove (différence entre la position actuelle et l'ancienne position)
Fonction appelé lorsque le widget est en train de bouger (x ou y modifié).
Si axis est égal à 0, l'axe modifié est celui des x. Sinon si il est égal à 1, l'axe modifié est celui des y.
Fonction dédiée à l'héritage et à la réimplentation qui est appelé par setX() ou setY().
Ajouté à la version 2.0.1.

_isNotFocusedAnymore()

Paramètres : aucun
Fonction appelé lorsque le widget n'est plus focus.
Fonction dédiée à l'héritage et à la réimplentation qui est appelé par MApp.
La fonction est concue pour que les modifications ne sont valables qu'une frame, il est donc conseillé, en cas d'utilisation de fonction "setQuelqueChose" de passer "False" comme second paramètre.
Ajouté à la version 1.0.0.

_isNotOverflightedAnymore()

Paramètres : aucun
Fonction appelé lorsque le widget n'est plus survolé par le curseur.
Fonction dédiée à l'héritage et à la réimplentation qui est appelé par MApp.
La fonction est concue pour que les modifications ne sont valables qu'une frame, il est donc conseillé, en cas d'utilisation de fonction "setQuelqueChose" de passer "False" comme second paramètre.
Ajouté à la version 2.0.0.

_isTextGettingEntered()

Paramètres : text (texte entrée)
Fonction appelé lorsque le widget est focus est que l'utilisateur rentre du texte.
Fonction dédiée à l'héritage et à la réimplentation qui est appelé par MApp.
La fonction est concue pour que les modifications ne sont valables qu'une frame, il est donc conseillé, en cas d'utilisation de fonction "setQuelqueChose" de passer "False" comme second paramètre.
Ajouté à la version 1.0.0.

_mouseMove()

Paramètres : buttons (touches de la souris clické), pos (nouvelle position de la souris), relativeMove (mouvement de la souris)
Fonction appelé lorsque le widget est focus est que l'utilisateur bouge la souris.
Fonction dédiée à l'héritage et à la réimplentation qui est appelé par MApp.
Ajouté à la version 1.3.0.

_mouseWheel()

Paramètres : rotation (valeur de la rotation de la molette)
Fonction appelé lorsque le widget est focus est que l'utilisateur tourne la molette.
Fonction dédiée à l'héritage et à la réimplentation qui est appelé par MApp.
Ajouté à la version 1.4.0.

_removeChild()

Paramètres : enfant (widget à supprimer en enfant)
Supprime l'enfant "enfant" de la liste d'enfant _children.
Modifié à la version 2.0.0.

_render()

Paramètres : aucun
Fait le rendu du widget et retourne le rendu final.
Si shouldModify est "true", la render normale à lieu. Sinon, il retourne _lastSurface.
Ajouté à la version 1.0.0.

_renderAfterHierarchy()

Paramètres : surface (surface parent)
Met la surface "surface" à jour avec le widget dessus et retourne "surface" mis à jour.
Cette fonction est appelée après le render de la hiérarchie dans _render. Il est fortement conseillé d'utiliser cette fonction dans l'héritage.
Modifié à la version 2.1.0.

_renderBeforeHierarchy()

Paramètres : surface (surface parent)
Met la surface "surface" à jour avec le widget dessus et retourne "surface" mis à jour.
Pour MWidget, il applique la couleur d'arrière plan backgroundColor.
Cette fonction est appelée avant le render de la hiérarchie dans _render.
Il est fortement conseillé d'utiliser cette fonction dans l'héritage.
Ajouté à la version 1.0.0.

_renderHierarchy()

Paramètres : surface (surface parent)
Met la surface "surface" à jour avec la hiérarchie du widget (les widgets enfants avec visible à True) et retourne "surface" mis à jour.
Cette fonction est appelée entre _renderBeforeHierarchy() et _renderAfterHierarchy().
Ajouté à la version 1.0.0.

Variables

backgroundColor

Couleur d'arrière plan du widget (avec le format RGB dans un tuple).
Ajouté à la version 1.0.0.

cursorOnOverflight

Icône du curseur lors du survol du widget.
L'icône doit être un icône de pygame.
Ajouté à la version 1.0.0.

focused

Booléen qui indique si le widget est le widget focus selon MApp.
Ajouté à la version 1.0.0.

height

Hauteur du widget.
Ajouté à la version 1.0.0.

ignoreUserEvent

Booléen qui indique si le widget ignore les évènements utilisateur.
Ajouté à la version 2.1.0.

mouseDown

Liste qui contient tous les boutons de la souris pressés par l'utilisateur sur le widget pendant cette frame.
Si bouton est "1", la touche clické est la touche gauche, si bouton est "2", la touche clické est la molette, si bouton est "3", la touche clické est la touche droite.
Si possible, il est conseillé d'utiliser _isGettingMouseDown.
Modifié à la version 2.0.0.

mouseUp

Liste qui contient tous les boutons de la souris plus pressés par l'utilisateur sur le widget pendant cette frame.
Si bouton est "1", la touche clické est la touche gauche, si bouton est "2", la touche clické est la molette, si bouton est "3", la touche clické est la touche droite.
Si possible, il est conseillé d'utiliser _isGettingMouseUp.
Modifié à la version 2.0.0.

overflighted

Booléen qui représente si le widget est survolé par le curseur ou non.
Si possible, il est conseillé d'utiliser _isGettingOverflighted.
Ajouté à la version 1.0.0.

parent

Parent du widget selon le système de hiérarchie.
Ajouté à la version 1.0.0.

shouldModify

Booléen qui indique si le widget doit se render pendant cette frame ou non.
Ajouté à la version 1.0.0.

visible

Booléen qui indique si le widget est visible ou non.
Ajouté à la version 1.0.0.

width

Largeur du widget.
Ajouté à la version 1.0.0.

x

Coordonnée x du widget.
Ajouté à la version 1.0.0.

y

Coordonnée y du widget.
Ajouté à la version 1.0.0.

_BACKGROUNDCOLOR

Variable (constante) égale à backgroundColor initial, modifiable que dans le constructeur et setBackgroundColor.
Ajouté à la version 1.0.0.

_children

Liste des enfants du widget.
Ajouté à la version 1.0.0.

_isMovingAtThisFrame

Booléen qui indique si le widget est en train de bouger (x ou y modifié) pendant cette frame.
Ajouté à la version 2.0.1.

_isResizedAtThisFrame

Booléen qui indique si le widget est en train de bouger (width ou height modifié) pendant cette frame.
Ajouté à la version 2.0.1.

_lastSurface

Surface correspondant à la dernière modification de la surface selon render.
Ajouté à la version 1.0.0.
